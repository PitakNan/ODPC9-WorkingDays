1. ภาพรวมระบบ

ระบบรายงานการมาปฏิบัติราชการ สคร.9 เป็นระบบที่ใช้ข้อมูลจากเครื่องสแกนลายนิ้วมือ นำมาประมวลผลและจัดทำรายงานเพื่อติดตามการมาปฏิบัติราชการของบุคลากร ครอบคลุม 25 กลุ่มงาน/หน่วยงานในสังกัด สคร.9

ข้อมูลปัจจุบัน: 325,217 รายการ จากบุคลากร 298 คน ครอบคลุมปีงบประมาณ 2564-2569

7. แนวทางจัดการข้อมูลปีเก่า (แก้ปัญหาขนาดไฟล์)

ปัญหา: Google Sheets จำกัดที่ 10 ล้าน cells ข้อมูลสะสมหลายปีทำให้เกินขีดจำกัด ต้องแยกไฟล์ทำให้ดูภาพรวมข้ามปีลำบาก

แนวทางแก้ไข

A. เก็บเฉพาะ 2 ปีล่าสุดใน Google Sheets

- Google Sheets เก็บแค่ปีงบปัจจุบัน + ปีก่อน (สำหรับ Report หน้า Sheets ปกติ)
- ข้อมูลปีเก่ากว่านั้น Back Up เป็น .xlsx แยกไฟล์ตามปีงบ
- Dashboard HTML รวมข้อมูลทุกปีได้ เพราะ embed JSON ไว้ในไฟล์เดียว

B. ใช้ Dashboard เป็นศูนย์กลางข้อมูลข้ามปี

- ทุกครั้งที่อัปเดต Dashboard ให้รวมข้อมูลจากทุกปี
- Dashboard จะเป็นที่เดียวที่ดูข้อมูลย้อนหลังได้ครบทุกปี
- ไฟล์ HTML เก็บข้อมูลได้ไม่จำกัด (ไม่มีปัญหา cell limit)

C. กระบวนการ Back Up ข้อมูล

เมื่อจบปีงบประมาณ (เดือนกันยายน): 1. Export ข้อมูลปีที่เก่ากว่า 2 ปี ออกจาก imp_data_Q 2. บันทึกเป็น "Data Scan XXXX.xlsx" (XXXX = ปีงบ) 3. เก็บทั้ง Raw Data และ Query Data ไว้ในไฟล์เดียวกัน 4. ลบข้อมูลเก่าออกจาก Google Sheets (เพื่อลดขนาด) 5. อัปเดต Dashboard ให้รวมข้อมูลทุกปี ตัวอย่าง โครงสร้างไฟล์: ├── Google Sheets (Online) │ └── Report Scan For User │ ├── imp_data_Q (ปีงบ 2568-2569) │ ├── imp_ref │ ├── Report_by dep(Q) │ └── Report_by person(Q) │ ├── Google Drive (Back Up) │ ├── Data Scan 2564.xlsx │ ├── Data Scan 2565.xlsx │ ├── Data Scan 2566.xlsx │ └── Data Scan 2567.xlsx │ └── Dashboard (Local/Shared) └── HR_Dashboard_สคร9.html (รวมทุกปี 2564-2569)

8. กระบวนการรายเดือน (Step-by-Step)

ทุกต้นเดือน (ภายในวันที่ 5)

1 Export ข้อมูลสแกน — ดึงข้อมูลดิบจากเครื่องสแกนลายนิ้วมือของเดือนที่ผ่านมา

2 นำเข้าข้อมูลดิบ — วางข้อมูลในไฟล์ Data Scan ปีปัจจุบัน (Sheet: DataXXXX)

3 รัน Query — ใช้ Query/สูตร เพื่อแปลงข้อมูลดิบเป็นรูปแบบมาตรฐาน (ชื่อ, PID, year, month, day, รหัสสถานะ)

4 นำเข้า imp_data_Q — Copy ผลลัพธ์จาก Query ไปต่อท้ายใน imp_data_Q ของ Google Sheets หลัก

5 ตรวจสอบ imp_ref — เช็คว่ามีบุคลากรใหม่หรือไม่ ถ้ามีให้เพิ่มใน imp_ref พร้อมระบุกลุ่มงาน

6 อัปเดต Dashboard — ดาวน์โหลด Report Scan For User เป็น .xlsx แล้วอัปโหลดให้ Claude สร้าง Dashboard ใหม่

7 ตรวจสอบรายงาน — เปิด Dashboard ตรวจสอบความถูกต้อง เปรียบเทียบกับ Report_by dep(Q) ใน Sheets

ทุกสิ้นปีงบประมาณ (กันยายน)

A Back Up ข้อมูลปีเก่า (ถ้าเกิน 2 ปีใน Sheets)
B สร้าง Dashboard สรุปประจำปี
C ทำความสะอาด imp_data_Q (ลบข้อมูลเก่าที่ Back Up แล้ว)

9. แนวทางพัฒนาในอนาคต

ระยะสั้น (สามารถทำได้ทันที)

- ใช้ Claude Scheduled Task ตั้งเตือนทุกต้นเดือนให้อัปเดตข้อมูล
- เพิ่มการ Export Dashboard เป็น PDF สำหรับรายงานผู้บริหาร
- เพิ่มตาราง Top 10 บุคลากรที่ขาด/สาย มากที่สุด

ระยะกลาง (1-3 เดือน)

- ใช้ Google Apps Script อัตโนมัติการ Query ข้อมูลดิบ
- เพิ่ม Dashboard แยกสำหรับข้อมูล WFH
- เพิ่มการแจ้งเตือนทาง Email เมื่อมีบุคลากรขาดงานติดต่อกัน

ระยะยาว (6-12 เดือน)

- ย้ายฐานข้อมูลไป SQLite/PostgreSQL เพื่อรองรับข้อมูลไม่จำกัด
- สร้าง Web Application สำหรับนำเข้าข้อมูลอัตโนมัติ
- เพิ่ม Machine Learning สำหรับ Predict แนวโน้มการลา/ขาด

สรุป: ระบบนี้ออกแบบให้ใช้งานง่าย ไม่ต้องติดตั้งซอฟต์แวร์เพิ่ม Dashboard เปิดในเบราว์เซอร์ได้ทันที และสามารถอัปเดตข้อมูลได้ทุกเดือนผ่าน Claude โดยแค่อัปโหลดไฟล์ .xlsx เข้ามา
